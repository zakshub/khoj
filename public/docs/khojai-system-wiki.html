KhojAI System Wiki

Architecture, Agents, Automation, and Operating Notes

Prepared for the Khoj STEAM School execution stack. Version 1.0, 31 May 2026.

Purpose

KhojAI is the automated work engine behind Khoj STEAM School. It is intended to generate educational content, run support agents, maintain a WhatsApp teacher experience, store conversations and lesson state, and publish outputs for the public Khoj product experience.

The public product experience should live at https://khoj.zuhaib.pro/. The internal tracking dashboard should remain separate at https://lab.zuhaib.pro/khoj.

Runtime Components

Next.js Website Serves the public KhojAI web experience. Package scripts define dev, build, and start on port 3001.
Scheduler Uses node-cron. Runs content pipeline every two days at midnight and support agents Monday and Thursday at 3 AM.
AI Teacher Uses WhatsApp Baileys session auth, Claude for replies, OpenAI Whisper for voice transcription, and SQLite tables for users and conversation history.
Legacy Meta Bot Express webhook implementation for Meta Cloud API. It is useful as a reference, but the active teacher path appears to be aiteacher/index.js.

Scheduler Behavior

Trigger Schedule Action
Content pipeline 0 0 */2 * * Runs runContentPipeline() every two days at midnight.
Support agents 0 3 * * 1,4 Runs runSupportAgents() every Monday and Thursday at 3 AM.
Manual pipeline node scheduler.js --now Immediately runs the content pipeline.
Manual support node scheduler.js --support Immediately runs support agents.

Database Notes

The confirmed local teacher code creates and uses these SQLite tables:

Table Purpose
wa_conversations Stores WhatsApp user and assistant message history.
wa_users Stores phone, name, relation, child profile, language, and message count.
conversations Used by the legacy Meta bot for webhook conversation history.
students Used by the legacy Meta bot for parent/student profile data.
content Referenced by both teacher paths for latest lesson context. Schema should be verified on VPS.

Operational Risks

Risk Why It Matters Recommended Fix
Auth/session files in local mirror WhatsApp Baileys auth files can expose session state. Keep aiteacher/auth/ out of Git and rotate sessions if exposed.
Hardcoded absolute VPS paths Local and VPS behavior can diverge. Move root paths into .env, with sane defaults.
Legacy and new bot both present Can confuse deployment and duplicate behavior. Mark bot/ as legacy or archive after verifying no process uses it.
Missing local full website and agents Harder to review or rebuild from local source. Sync VPS source into repo, excluding secrets, logs, database, and generated outputs.
